opencv windows11下opencv4.8.0-cuda C++&Python版本源码编译

前期准备

cmake、vs2022、opencv4.8.0、opencv_contrib-4.8.0、cuda、cudnn、python3

安装CUDA及CUDNN

参考链接：https://blog.csdn.net/yangyu0515/article/details/129643486

安装Cmake

Cmake下载地址：https://cmake.org/download/ 安装系统对应版本，安装时添加到环境。

下载Opencv及Opencv_contrib源码

Opencv源码下载地址：https://opencv.org/releases/

Opencv_contrib源码下载地址：https://github.com/opencv/opencv_contrib/tags

选择opencv对应的contrib版本

安装原生Python3

在使用 CMake 为 Anaconda/Miniconda 新建的虚拟环境安装 OpenCV 时，需要在主机上安装一个与虚拟环境中的 Python 版本一致的原生环境，这是由于CMake 需要在构建过程中使用与虚拟环境中的 Python 版本一致的 Python 解释器来生成适用于该版本的 Python 绑定。

安装时勾选加入系统环境，安装后给python安装numpy。

Python下载地址：https://www.python.org/downloads/

开始编译

Cmake编译

  1. 打开CMake，where is the source code是Opencv sources的文件夹位置，where to build the binaries是编译opencv保存的文件夹位置(自定义)，在左下角第一次点击Configure。

  2. 选择对应vs的版本，系统选择x64，最后点右下角Finish。

  3. 显示Configuring done：

    • 安装到默认环境(base)不需要执行此步骤，该步骤步骤的目的是安装cuda版本opencv到用户自定义的虚拟环境中，分别将路径指向自定义虚拟环境的对应位置 : PYTHON3_EXECUTABLE、PYTHON3_INCLUDE_DIR、PYTHON3_LIBRARY、PYTHON3_NUMPY_INCLUDE_DIRS(需要安装numpy)、PYTHON3_PACKAGES_PATH。
    • 在Search搜索框搜索带cuda的关键字，全部勾选。
    • 在Search搜索框搜索OPENCV_EXTRA_MODULES_PATH，添加解压的opencv_contrib中的modules的路径。
    • 搜索并勾选OPENCV_ENABLE_NONFREE选项，
    • 搜索并勾选带python的选项。
    • 搜索并取消勾选不想编译的Java，Jscript，test选项。
    • 搜索并勾选build_opencv_world可以将所有opencv的库都打包编译在一起，后续配置C++的opencv环境时不需要自己手动选择添加每个小模块，但是具体的工程又不建议这种模式。
    • 再次Configure

  4. 显示Configuring done后，将CUDA_ARCH_BIN显卡算力内容改成自己显卡的算力。官网查看显卡算力地址，删除小于自己显卡算力的部分，再搜索并勾选ENABLE_FAST_MATH选项，第三次点击Configure。

  5. 显示Configuring done后，点击generate，显示generating done，成功完成cmake编译，在输出的opencv保存文件夹位置中存在就生成了OpenCV.sln文件。

VS2022编译

  1. 使用VS2019 (以管理员方式运行) 打开刚刚编译工程OpenCV.sln，在release|x64模式下，
  2. 确保在解决方案的bindings这个目录下有opencv_python3，否则即使编译成功仍然不能使用CUDA加速。
  3. 在解决方案资源管理器—>CMakeTargets—>ALL_BUILD–>右键点击生成。
  4. 同样在release|x64模式下，在解决方案资源管理器—>CMakeTargets—>INSTALL–>右键点击生成。

编译完成后在where to build the binaries位置的install文件夹内生成以下内容。

可能出现的问题

cmake编译过程中可能出现的问题

  1. cmake编译出现Download failed的问题。在where to build the binaries位置里面有个CMakeDownloadLog.txt，将里面下载链接复制到浏览器进行下载，将其和where is the source code里面的.cache文件夹里面内容相对应，下图是下载失败时候，文件大小是0KB，手动下载完成后进行替换。

  2. 注意，用来替换的文件的名称要与对应空文件的名称保持一致。

  3. 对于部分文件则需要将网页以另存为的方式进行替换，注意一定不要以复制网页内容粘贴到空文件的方式进行替换，这是无效的的。

  4. cmake configure后始终没有出现BUILD_opencv_python3的选项，很可能是Python3的环境变量没有配置好，在第一次configure时没有识别成功，可以在环境变量中配置好python3的PYTHON3_EXECUTABLE、PYTHON3_INCLUDE_DIR、PYTHON3_LIBRARY、PYTHON3_NUMPY_INCLUDE_DIRS(需要安装numpy)、PYTHON3_PACKAGES_PATH。再重新configure。

VS2022编译过程中可能出现的问题

  1. 通常是对于部分资源，vs2022没有管理员操作权限，因此只需要用管理员身份重新打开进行操作即可。（暂时没遇到这个问题）
  2. 在实际编译中遇到两个小问题，在百度后得以解决，这两个问题都对源码进行了小改动。

安装编译好的Opencv Python版

在编译完成不报错的情况下，opencv已经安装在了Python环境中，pip list无法识别到opencv-python 但是import cv2可以正常使用。

如果需要在虚拟环境中安装，需要把原生Python环境中Lib\site-packages下的cv2文件夹拷贝到虚拟环境中的Lib\site-packages下，并把编译成功后的文件夹下的build\lib\python3\Release\cv2.cp311-win_amd64.pyd复制到虚拟环境中的Lib\site-packages文件夹下改名成 cv2.pyd。
